    61→    61→### 关键技术决策：daemonize.py 守护进程
    62→    62→**问题**: 沙箱环境中使用 `nohup ... &` + `disown` 启动的 Next.js dev server 会在父 shell 退出时被 SIGKILL，导致每次 bash 命令结束后服务器都死掉。`setsid`、`nohup`、`disown`、keep-alive 脚本均无法解决（父 shell 退出时整个进程组被回收）。
    63→    63→
    64→    64→**解决方案**: 编写 `/home/z/my-project/daemonize.py`，使用经典 double-fork daemon 模式：
    65→    65→1. 第一次 `os.fork()` → 父进程退出
    66→    66→2. `os.setsid()` → 创建新会话，脱离控制终端
    67→    67→3. 第二次 `os.fork()` → 子进程退出，孙进程被 init (PID 1) 收养
    68→    68→4. 重定向 stdin/stdout/stderr 到 `/dev/null` 和 `dev.log`
    69→    69→5. `os.execvpe("next", ["next", "dev", "-p", "3000"], env)` 替换进程映像
    70→    70→
    71→    71→启动命令：
    72→    72→```bash
    73→    73→python3 /home/z/my-project/daemonize.py next dev -p 3000
    74→    74→```
    75→    75→
    76→    76→进程 PID 5025 (`next-server`)，PPID 为 1（init），完全脱离调用 shell，在 bash 命令之间稳定存活。
